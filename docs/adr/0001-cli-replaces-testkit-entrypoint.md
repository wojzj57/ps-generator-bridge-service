# CLI replaces the testkit entrypoint

We will migrate the Photoshop smoke harness out of `@ps-generator-bridge/testkit` and into `@ps-generator-bridge/cli`, without preserving the old `ps-bridge-test` binary or the old `packages/testkit` workspace package. This is a deliberate breaking refactor: the new CLI package becomes the single command-line surface for current smoke checks and future local tooling, rather than carrying a compatibility alias that keeps the old testkit package identity alive. The binary is `ps-generator-bridge`; existing smoke commands become top-level `ps-generator-bridge run` and `ps-generator-bridge dev`, while the current generator-core preparation command becomes `ps-generator-bridge setup-core` so future broader setup/install workflows can use `ps-generator-bridge setup`. The package only commits to the CLI binary, not to importable harness APIs. The generator-core checkout lives at `./generator-core` under the nearest ancestor containing `pnpm-workspace.yaml`; outside a workspace it falls back to `AppData\Local\Temp\ps-generator-bridge\generator-core`.
