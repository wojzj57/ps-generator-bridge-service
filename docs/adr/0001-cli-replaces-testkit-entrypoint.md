# CLI replaces the testkit entrypoint

The Photoshop smoke harness lives in `@ps-generator-bridge/cli`; the removed `@ps-generator-bridge/testkit` package and `ps-bridge-test` binary have no compatibility aliases. This deliberate breaking refactor makes the CLI package the single command-line surface for smoke checks and local tooling. Its binary is `ps-generator-bridge`; smoke commands are `run` and `dev`, generator-core preparation is `setup-core`, and generator runtime installation is `setup`. The package commits only to the CLI binary, not to importable harness APIs. The generator-core checkout lives at `./generator-core` under the nearest ancestor containing `pnpm-workspace.yaml`; outside a workspace it uses a stable per-user cache directory: `%LOCALAPPDATA%\ps-generator-bridge\generator-core` on Windows, `~/Library/Caches/ps-generator-bridge/generator-core` on macOS, or `$XDG_CACHE_HOME/ps-generator-bridge/generator-core` on Linux (falling back to `~/.cache`).
