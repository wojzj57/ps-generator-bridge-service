# Release Process

This repository publishes three npm packages from the monorepo:

- `@ps-generator-bridge/sdk`
- `@ps-generator-bridge/generator`
- `@ps-generator-bridge/testkit`

The private root package is not published. The three published packages use a unified version through Changesets.

## Authoring a releasable change

For changes under package runtime sources, add a changeset before opening or updating the PR:

```sh
pnpm changeset
```

The PR changeset check requires a changeset when files under these paths change:

- `packages/sdk/src/`
- `packages/generator/src/`
- `packages/generator/jsx/`
- `packages/testkit/src/`

If a PR touches those paths but should not publish a package version, apply the `no-release` label to the PR.

## Version PR

Merging releasable changes to `main` lets the release workflow create or update a Changesets version PR. That PR updates package versions, updates changelogs when generated by Changesets, and consumes pending changeset files.

## Publishing

After the version PR is merged to `main`, the release workflow runs these gates before publishing:

```sh
pnpm format:check
pnpm typecheck
pnpm test
pnpm build
pnpm pack:check
```

If all gates pass, the workflow publishes the three packages to npm with the `latest` dist-tag.

## npm Trusted Publishing

Publishing uses npm Trusted Publishing / OIDC, not a long-lived `NPM_TOKEN`. Configure each npm package with a trusted publisher that points at this GitHub repository and workflow:

- Repository: `wojzj57/ps-generator-bridge-service`
- Workflow file: `.github/workflows/release.yml`
- Packages:
  - `@ps-generator-bridge/sdk`
  - `@ps-generator-bridge/generator`
  - `@ps-generator-bridge/testkit`

The workflow has `id-token: write` permission so npm can verify the GitHub Actions identity.
